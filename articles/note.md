# 10年分のEvernote 11,000ノートを、Obsidianへ無傷で引っ越した話

Evernoteをずっと使ってきた。気づけば**11,238ノート、添付ファイル込みで16GB**。
重くなったし、そろそろ自分の手元（プレーンなMarkdown）に置き直したくて、**Obsidian**へ全部移すことにした。

結論から言うと、移行そのものは既存の優秀なツール2つでほぼ片づく。けれど「**大量にある**」「**保存先をiCloudにした**」「**ノート名が日本語**」——この3つが重なると、ガイドに載っていない壁が次々に出てくる。同じ条件で引っ越す人のために、つまずいた6か所を残しておく。

（手順とスクリプトはGitHubにMITで置いた：https://github.com/akihidem/evernote-to-obsidian-largevault ）

## 全体の流れ

やることは3ステップだけ。

1. **Evernoteからアカウントを丸ごとダウンロード**（`evernote-backup`というツール）
2. **Markdownに変換**（`yarle`というツール）
3. **Obsidianの保管庫（Vault）に置いて、ちゃんと全部あるか検証**

絵にするとこう。

```
Evernote ──全ダウンロード──▶ 変換 ──▶ Obsidian（iCloud上のVault）
```

シンプルに見える。が、ここからが本番だった。

## つまずき① 1つのノートブックが「9.8GB」になって変換ツールが死ぬ

最初、ノートブック単位で書き出したら、あるファイルが**9.8GB**になっていた。変換ツール（yarle）はファイルを**まるごと一度メモリに読み込む**設計なので、これを開いた瞬間に限界を超えて落ちる。1ノートも変換できない。

> 解決：「1ノート＝1ファイル」に分割して書き出す（`--single-notes`というオプション）。さらに**ノートブックごとに**変換を回す。こうすると1個1個が小さくなり、安全に流れる。

## つまずき② iCloudにコピーしたら、ノートが「消えた」ように見える

変換後、Vaultには11,168個のノートが入っているはず。なのに数えると**11,102個**しかない。血の気が引いた。

正体は、**iCloud（Windows版）のアップロードの仕様**だった。iCloudはクラウドに送信中のファイルを、いったん `.元の名前.ランダム6文字` という**隠しファイル名**に変える。送信が終わると元に戻す。つまり送信中は「見た目上いなくなる」だけで、**中身は完全にディスク上にある**（サイズが原本と1バイトも違わないことを確認した）。

> 解決：その隠し名を元の名前に戻すだけ。プログラムで一括リネーム。安全（消さない・既にある名前は触らない）。iCloudは大量アップロードの間ずっと隠し名を作り続けるので、何度か実行する。

ここは本当に怖かった。**「消えた」と「隠れている」はまったく違う**。慌てて消したり再コピーしたりせず、まず正体を突き止めたのが正解だった。

## つまずき③ 名前に「.app」が入ったノートだけフォルダが化ける

`Mac写真.appの重複一括削除` というノートだけ、フォルダ名が `Mac写真の重複一括削除(1).app` に勝手に変わっていた。iCloudが「`.app` ＝ Macのアプリ」と勘違いして再編してしまう仕様。中身は無事で、**フォルダ名だけ**の問題。

> 解決：フォルダ名から `.app` を消す（`_app` に置換）。ドットが無くなればiCloudはもう触らない。

## つまずき④ 変換ツールの地味な罠2つ

- オプションは `--config` ではなく **`--configFile`**。間違えると黙って無視され、別物を読んで落ちる。
- テンプレートファイルを**実行した場所基準**で探すので、隣に置いておく必要がある。

（細かいが、ここで30分溶かした人は多いはず）

## つまずき⑤ 量が多いと、ダウンロード途中で止められる

Evernoteには**APIの利用制限**がある。1万件もあると途中で「○分後に再開して」と止められる。幸い**続きから再開できる**ので、止められたら待って自動で再開、を繰り返すようにした。1万件のダウンロードは、数回に分かれて完了した。

## つまずき⑥ 最後は「目視」ではなく「数」で無傷を証明する

ここまで散々リネームしているので、ぱっと見では信用できない。そこで**移行元と移行先の総ファイル数をプログラムで突き合わせ**た。日本語の文字は、見た目が同じでも内部コードが2種類ある（NFC／NFD）ため、そこも揃えてから比較する。

最終結果：

```
移行元  45,221ファイル
移行先  45,221ファイル   → 完全一致
```

これでようやく「1つも失っていない」と言い切れた。

## おまけ：1万ノートの整理は「やらない」のが正解

全部移したあと、「これをどう整理するか」で手が止まる。が、Obsidian界隈の定石はむしろ逆だった。

> **手で振り分けない。** 取り込んだ1万件は丸ごと「アーカイブ」に寝かせ、その上に**薄い入れ物**（Projects / Areas / Resources / Archive ＝ PARA法）だけ作る。普段はフォルダを掘らず、**全文検索**と、テーマごとのリンク集（MOC）で必要な分だけ引き出す。

1万件を1件ずつ仕分ける人生はない。**寝かせて、検索で起こす**。これがいちばん効いた。

## おわりに

引っ越し自体は半日。けれど「消えたように見えるけど消えていない」を見破れるかどうかで、結末がまるで変わる。**慌てて消す前に、まず正体を数える**。これは引っ越しに限らず、データを触るとき全部に効く教訓だった。
